Notification Service
Developer Documentation

A FastAPI-based appointment reminder system that syncs from Google Calendar and ClickUp, sends SMS/email notifications at 72h / 24h / 2h windows, processes customer replies, and surfaces everything through a staff dashboard.

Email — Queue-Based

→
INSERT email_queue
→
cron: 5,35 * * * *
→
sendgrid_email.py
→
SendGrid API
→
event webhook
Why queue emails? Decouples engine from send latency, allows retries (max 3 attempts), and prevents engine stalls on SendGrid outages.

Deduplication

A UNIQUE constraint on (appointment_id, rule_name, channel, appointment_at) means the cron can safely run multiple times — the second INSERT silently fails.

Template Matching

Template lookup uses a fallback chain — most specific to least specific:

channel + specific_type + specific_lang + rule
Exact match
channel + all + specific_lang + rule
Type fallback
channel + all + en + rule
Language fallback
No match
Log WARNING, skip send

Workers & Cron Jobs

*/15 * * * *
Calendar Sync
calendar_sync.py
Fetches events from Google Calendar (service account), extracts customer name/phone/language, upserts into appointments table. Quarantines records with missing/invalid data.
*/30 * * * *
Notification Engine
notification_engine.py
Evaluates all upcoming appointments against notification rules. Checks kill switch, quiet hours, opt-outs, dedup. Sends SMS synchronously; queues emails to email_queue.
5,35 * * * *
Email Worker
channels/sendgrid_email.py
Drains email_queue (sent_at IS NULL, attempts < 3). Renders template + replaces [LOGO_BANNER]. Sends via SendGrid. Max 3 retries per row.
*/5 * * * *
Reply Processor
workers/reply_processor.py
Classifies unprocessed inbound SMS. Escalates reschedule/question/unknown replies to Discord, staff SMS, and staff email. Marks rows as processed.

Webhook Routes

Unauthenticated. All Twilio routes validate X-Twilio-Signature. ClickUp validates X-Signature (HMAC-SHA256).

POST
/notifications/webhooks/twilio/sms
Inbound customer SMS. Logs to inbound_messages, handles STOP/START opt-outs, fires Discord alert.
POST
/notifications/webhooks/twilio/status
Twilio delivery status. Updates notification_attempts.status (delivered, failed, undelivered…).
POST
/notifications/webhooks/sendgrid/status
SendGrid event array. Parses sg_message_id, updates email_queue + notification_attempts.
POST
/notifications/webhooks/clickup
ClickUp task update. Validates signature, extracts fields, upserts appointment or quarantines. Always 200.
GET
/health
Returns {"status": "ok"}. Used by uptime monitors.
⚠ SendGrid signature validation not implemented. The SendGrid status webhook currently accepts all POSTs without verifying the signature. This should be addressed before the service handles high volumes.

Auth & Sessions

Session Mechanism

Signed itsdangerous.URLSafeTimedSerializer cookie containing {email, role}. Cookie is HttpOnly, SameSite=Lax, Secure (production).

Default max age: 8 hours (SESSION_MAX_AGE_SECONDS=28800).

Rate Limiting

5 failed login attempts per IP per 15-minute window. Cleared on successful login. In-memory only (resets on server restart).

ClickUp Integration

ClickUp fires a webhook on task update. The service validates the HMAC-SHA256 signature, extracts customer fields using configured field UUIDs, and upserts an appointment.

Field ID Configuration

Run python get_fields.py to discover the field UUIDs for your ClickUp list. Then set them in .env:

CLICKUP_FIELD_NAME=1ca299a3-2a6a-4102-b884-83d0ecaaa9c3   # Full Name
CLICKUP_FIELD_PHONE=dd2505b0-b8ef-43cb-a730-4193efe7b664  # Phone
CLICKUP_FIELD_EMAIL=333855a2-a98b-4fe9-b951-a9fd075c46d1  # Email
CLICKUP_FIELD_SCOPE=50870f6d-1fa0-4cbc-af66-119a3de6d4b7  # Scope Of Work (multi-select)
CLICKUP_FIELD_DATE_HVAC=8454f592-...
CLICKUP_FIELD_DATE_SOLAR=e55a9e51-...
# …one date field per service type

Scope → Date Field Mapping

Each scope-of-work value maps to its relevant date field. First match wins when a task has multiple services.

Idempotency

Each webhook event is recorded in the webhook_events table with a UNIQUE constraint on (source, external_event_id). Duplicate webhook deliveries are safely ignored.

Design Rules

R1
Always use get_connection()Never call sqlite3.connect() directly. get_connection() enables WAL mode, busy timeout, foreign keys, and row_factory.
R2
Credentials from .env onlyNever hardcode keys, tokens, passwords, or SIDs. Never log these values.
R3
Validate all webhook signaturesTwilio: X-Twilio-Signature. ClickUp: X-Signature (HMAC-SHA256). SendGrid: ⚠ not yet implemented.
R4
Track every notification attemptInsert into notification_attempts before sending. Update status after send or receipt.
R5
Respect opt-outsCheck opt_outs table before every SMS send. STOP words auto-create opt-out rows.
R6
Never send email inlineAlways queue to email_queue. The engine never calls SendGrid directly.
R7
UNIQUE constraint deduplicationnotification_attempts has a UNIQUE key on (appointment_id, rule_name, channel, appointment_at). Safe to run engine multiple times.
R8
Quarantine bad dataValidate phone, name, date. Quarantine instead of silently skipping. Surface to staff for resolution.
R9
Enforce quiet hoursDon't send outside quiet_hours_start…quiet_hours_end. Configurable per timezone. Check before every engine run.
R10
Bind to 127.0.0.1:8096Never expose FastAPI directly to the internet. Always terminate TLS at Nginx.
R11
Respect the kill switchCheck notifications_paused setting before all engine work. If TRUE, exit immediately.
